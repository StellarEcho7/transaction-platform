## When to use me

Before implementing anything new — especially when about to write "yet another" function or unsure if similar logic exists. Ensures consistency and prevents duplication.

## Output

1. Relevant existing code (files, modules, patterns)
2. Reuse opportunities vs new implementation recommendation
3. Risks (duplication, inconsistency)
4. Next step (specific action)
